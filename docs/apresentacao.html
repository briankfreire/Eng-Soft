Plataforma de Inovação Colaborativa

Microsserviço de Colaboradores

  • Engenharia de Software - Mackenzie
  • Arquitetura de microsserviços + IA
  • 4 serviços Flask + Frontend integrado

Visão Geral

Objetivo da Plataforma

  • Conectar idealizadores e colaboradores em projetos de inovação
  • Facilitar formação de equipes multidisciplinares
  • Usar IA para enriquecer perfis e melhorar matchmaking

Nosso Papel

  • Lado "Colaboradores" da plataforma
  • Autenticação, perfil, competências e projetos
  • IA para enriquecimento de perfil (bio automática)

Funcionalidades Principais

  • Cadastro e Login: autenticação de colaboradores
  • Perfil Completo: nome, bio, foto, disponibilidade, links
  • Competências (Skills): cadastro com níveis de proficiência
  • Descoberta de Projetos: buscar projetos da turma
  • Meus Projetos: vincular-se e ver projetos participantes
  • IA - Gerar Bio: bio profissional baseada em skills

IA - Enriquecimento de Perfil

Tipo de IA Escolhido

  • Enriquecimento de Perfil (conforme enunciado)
  • IA ajuda a escrever bio de maior impacto

Como Funciona

  • Backend busca skills cadastradas do usuário
  • Monta prompt em português com as competências
  • Chama API Groq (llama-3.3-70b-versatile)
  • Retorna bio curta que o usuário pode ajustar

Microsserviço Auth

Responsabilidade

  • Autenticação e diretório de usuários
  • Dados: email, senha (hash), timestamps

Principais Endpoints

  • POST /register - criar usuário
  • POST /login - autenticar e retornar user.id
  • GET /users?email= - buscar por email
  • GET /users/list - listar todos
  • GET /metrics - estatísticas

Microsserviço Profile

Responsabilidade

  • Perfil completo do colaborador
  • Links externos e indicador de completude

Principais Endpoints

  • PUT /profiles/{user_id} - criar/atualizar
  • GET /profiles/{user_id} - obter perfil + links
  • POST /profiles/{user_id}/links - adicionar link
  • POST /profiles/{user_id}/bio/suggest - IA gerar bio
  • GET /metrics - estatísticas de perfis

Microsserviço Skills

Responsabilidade

  • Catálogo de competências por usuário
  • Dados: skill_name, proficiency (basic/intermediate/advanced)

Principais Endpoints

  • POST /users/{user_id}/skills - adicionar skill
  • GET /users/{user_id}/skills - listar skills
  • DELETE /users/{user_id}/skills/{id} - remover

Integração com IA

  • Profile Service consulta este serviço para montar prompt da IA

Microsserviço Projects

Responsabilidade

  • Ligação colaborador ↔ projeto
  • Integração com banco de projetos da turma

Endpoints - Projetos

  • GET /proxy/projects - lista projetos (com busca)
  • POST /projects/{id}/collaborators/{user_id} - vincular
  • GET /projects/{id}/collaborators - membros
  • GET /collaborators/{user_id}/projects - projetos do user

Projects - Agregação para Outros Grupos

Endpoints de Colaboradores

  • GET /collaborators/search?email=|user_id=
  • Retorna: user + profile + skills (pacote completo)

Listagem Paginada

  • GET /collaborators?page=&page_size=
  • Lista colaboradores com dados resumidos
  • Permite outros grupos fazerem matchmaking

Frontend de Colaboradores

Tecnologia

  • HTML + Tailwind CSS (estático)
  • Consome os 4 microsserviços em Azure

Fluxo Principal

  • Login → carregar perfil e skills
  • Preencher/ajustar perfil, adicionar links
  • Clicar "Gerar Bio com IA" para sugestão personalizada
  • Buscar projetos, vincular-se, ver "Meus Projetos"
  • Botão de atalho para app Idealizador

Integrações com Outros Grupos

Consumimos

  • Banco de Projetos da turma: bdprojetos.azurewebsites.net
  • API Groq: llama-3.3-70b-versatile para IA

Oferecemos

  • Endpoints de colaboradores (search/list) no Projects Service
  • Outros grupos podem buscar e listar colaboradores
  • Integração bidirecional: nossa API é parte da plataforma única

Arquitetura da Solução

Backend

  • 4 microsserviços Flask independentes
  • SQLite em cada serviço
  • CORS habilitado

Deploy

  • Azure Web Apps (Python 3.12)
  • GitHub Actions CI/CD
  • Gunicorn + porta 8000

Frontend

  • HTML estático + Tailwind
  • GitHub Pages
  • Consume APIs diretamente

Local

  • Docker Compose
  • Nginx para microfrontends
  • 4 containers + frontend

Resumo

Entregamos

  • ✓ Microsserviços robustos e independentes
  • ✓ IA funcional para enriquecimento de perfil
  • ✓ Integração com banco de projetos da turma
  • ✓ APIs abertas para outros grupos consumirem
  • ✓ Frontend completo e funcional em Azure
  • ✓ CI/CD automatizado com GitHub Actions
